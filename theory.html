Maximum Likelihood Decoding of Linear Codes on Binary-Input Memoryless Channels

As described in the theory of the previous experiment, a memoryless channel is described by the input alphabet X\cal XX (set of possible values taken by the input random variable XXX), the output alphabet Y\cal YY (possible values taken by the output random variable YYY), and the transition probabilities pY∣X(y∣x),∀x∈X,y∈Yp_{Y|X}(y|x), \forall x\in{\cal X}, y\in{\cal Y}pY∣X​(y∣x),∀x∈X,y∈Y.

Consider that we use the channel nnn times, i.e, our input sequence is a vector x=(x1,…,xn)∈Xn\bm{x}=(x_1,\ldots,x_n)\in {\cal X}^nx=(x1​,…,xn​)∈Xn (recall that Xn{\cal X}^nXn denotes the set of all nnn-length tuples with entries from X{\cal X}X). Because of the noisy characteristics of the channel (governed by the transition probabilities pY∣Xp_{Y|X}pY∣X​), the transmitted input vector x\bm{x}x is transformed, into a random output vector Y=(Y1,…,Yn)\bm{Y}=(Y_1,\ldots,Y_n)Y=(Y1​,…,Yn​), governed by the transition probabilites pY∣X(y∣x)=∏i=1np(yi∣xi)p_{\bm{Y}|\bm{X}}(\bm{y}|\bm{x})=\prod_{i=1}^np(y_i|x_i)pY∣X​(y∣x)=∏i=1n​p(yi​∣xi​).

The receiver observes the output sequence Y=y\bm{Y}=\bm{y}Y=y. The goal of the receiver is then to decode the transmitted vector x\bm{x}x. The estimate of the transmitted vector is denoted by x^\hat{\bm{x}}x^. However, because of the fact that the channel noise is random, there could be several possible vectors in Xn{\cal X}^nXn which result in the same received vector y\bm{y}y. Therefore, the decoder at the receiver has to have some metric, based on which it can decide the value for the estimate x^\hat{\bm{x}}x^.

Specifically, we consider the likelihood as the decoding metric, in this virtual lab. That is, the decoder seeks to find that x\bm{x}x which maximizes the probability p(y∣x)p(\bm{y}|\bm{x})p(y∣x). x^=argmax⁡p(y∣x).\hat{\bm{x}}=arg\max p(\bm{y}|\bm{x}).x^=argmaxp(y∣x). The probability p(y∣x)p(\bm{y}|\bm{x})p(y∣x) is called the likelihood of x\bm{x}x with respect to the received vector y\bm{y}y.

However, the problem that there could be several candidates for such x∈Xn\bm{x}\in{\cal X}^nx∈Xn which maximize the likelihood. This problem can be resolved by choosing the transmit sequences from a code. Specifically, we are interested in choosing nnn-length linear codes for binary input channels.

Using Linear Codes on Memoryless Binary-Input Channels

As defined before, an nnn-length linear code C\cal CC over F2\mathbb{F}_2F2​ of rate kn\frac{k}{n}nk​ is a kkk-dimensional subspace of F2n\mathbb{F}_2^nF2n​.

The ML Decoding Rule for Linear Codes

Assuming that the receiver receives the vector y\bm{y}y, the Maximum Likelihood decoding rule (called the ML decoding rule) when using such a code on a binary-input channel, is written as follows.

x^=argx∈Cmax⁡p(y∣x).\hat{\bm{x}}=arg_{\bm{x}\in{\cal C}}\max p(\bm{y}|\bm{x}).x^=argx∈C​maxp(y∣x).

In case of there being multiple codewords in C\cal CC which maximize the likelihood, we will assume that the decoder will break the ties arbitrarily, i.e., the decoder declares any of these codewords (that maximize the likelihood) as the estimate. This estimate x^\hat{\bm{x}}x^ is called the ML estimate for the input codeword x\bm{x}x.

3. AWGN Channel with Noise N(0,N0/2){\cal N}(0,N_0/2)N(0,N0​/2):

Now, what does it mean to use a binary code on this channel? Observe that the channel really accepts only real numbered input symbols, however, we have binary digits (bits) in the codeword. Therefore, we need a modulation scheme, which maps the bits to real-numbered symbols that we can transmit.

The simplest modulation scheme when we use a binary code (for example the code C\cal CC above) on the AWGN channel is to map the binary symbols {0,1}\{0,1\}{0,1} to {+1,−1}∈R\{+1,-1\}\in\mathbb{R}{+1,−1}∈R respectively. This essentially translates to using a BPSK (Binary phase shift keying) modulation scheme on the AWGN channel, to transmit the bits of the codeword. The fact that we assume bit 000 is mapped to +1∈R+1\in\mathbb{R}+1∈R and the bit 111 is mapped to −1∈R-1\in\mathbb{R}−1∈R translates to using SNR (signal-to-noise ratio) per codeword-bit as 1/(N0/2)1/(N_0/2)1/(N0​/2). To mathematically model a larger SNR value, we could then assume that the channel has a smaller noise power N0/2N_0/2N0​/2, without having to change the mapping of the BPSK modulation itself. Under this modulation scheme, the code C\cal CC is effectively written as follows. C={x1=(1,1,1,1,1),x2=(−1,1,−1,1,−1),x3=(1,−1,1,−1,−1),x4=(−1,−1,−1,−1,1)}.{\cal C}=\{\bm{x}_1=(1,1,1,1,1),\bm{x}_2=(-1,1,-1,1,-1), \bm{x}_3=(1,-1,1,-1,-1), \bm{x}_4=(-1,-1,-1,-1,1)\}.C={x1​=(1,1,1,1,1),x2​=(−1,1,−1,1,−1),x3​=(1,−1,1,−1,−1),x4​=(−1,−1,−1,−1,1)}.
